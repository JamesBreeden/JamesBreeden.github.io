The Button

This project was dubbed "The Button" because of how simple it seemed. When it began, it was a simple WinForm that had a single button. It developed into much more than that.

What the team requested was a more secure way for the CCT team to access batch files. What I created was an application that can access the batch files, error checks, and reports the result back to the user. If there were any errors, several events occur.

First event is the error gets written to a .txt file on the support's server. Second event is the user is give the opportunity to communicate with support staff via a Yes/No dialog box. If the user selects yes, it will auto-populate an email with target, subject, and content to be sent to support staff. If the user selects no, the program will go back to home screen without emailing support.

The error checking not only makes sure the batch file was successful, but it also makes sure it exists. By validating the existance of the batch file, it helps with the user experience as well as helps support with troubleshooting.

One final feature I added in was the Konami Code. I captured the key input from the keyboard, and if it matched up to the required sequence, the text on the button would change. The text started out as a professional string such as "Execute Batch File," but when the Konami Code is entered, the program access an external .txt file, reads in the lines, and uses a random number generator to select one of the lines and display it as the text on the button.

In the .txt file are various quotes from many different sources. Some were from movies, some were from cartoons, and some were from around the office. All of this was done AFTER getting authorization ofcourse.

One of the more... imaginative projects, was creating a golf putting practice target. Sometimes, to reset their brain and come with a different approach, a person needs a distraction. Using a cardboard box, scissors, and some clever thinking, I created a multi-target practice target to use.
